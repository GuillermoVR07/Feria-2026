### Huecos detectados y corregidos localmente

Se detectaron estos eventos obligatorios sin auditoria funcional explicita:

- `CONSENT_ACCEPTED`
- `RECOMMENDATION_CREATED`
- `AI_INFERENCE_FAILED`

Archivos modificados:

- `supabase/functions/create-case/index.ts`
- `supabase/functions/run-inference/index.ts`

Cambios aplicados:

1. `create-case`
   - Ahora recupera `consent_records.id` al insertar consentimiento.
   - Registra evento `CONSENT_ACCEPTED` con:
     - `entity_type = consent_records`
     - `entity_id = consentRecord.id`
     - `case_id`
     - `case_code`
     - `consent_version`

2. `run-inference`
   - Ahora recupera `recommendations.id` al insertar recomendacion.
   - Registra evento `RECOMMENDATION_CREATED` con:
     - `entity_type = recommendations`
     - `entity_id = recommendationRecord.id`
     - `case_id`
     - `inference_id`
     - `suspicion_level`
     - `urgency_level`
     - `professional_referral`
     - `reason_codes`

3. `run-inference`
   - Si ocurre `AI_INFERENCE_FAILED`, ademas de mover el caso a `ai_failed`, registra evento `AI_INFERENCE_FAILED`.
   - Esto aplica solo cuando hubo una falla de inferencia real.
   - La falta de modelo activo sigue devolviendo `AI_SERVICE_UNAVAILABLE`, porque no hay servicio IA operativo.

## Fase 13 - Validaciones obligatorias

### Validaciones cubiertas por base de datos

La migracion `0002_tablas_base.sql` ya cubre:

- `age_years` entre 0 y 120 o nulo.
- `city` maximo 120 caracteres.
- `zone` maximo 120 caracteres.
- `lesion_duration_days` entre 0 y 3650.
- `consent_records.accepted = true`.
- `risk_questionnaires.notes` maximo 1000 caracteres.
- `specialist_reviews.clinical_notes` maximo 5000 caracteres.
- `ai_inferences.probability` entre 0 y 1.
- `case_images.mime_type` restringido a imagenes permitidas y PDF.
- `case_images.size_bytes` mayor a 0 y maximo 10 MB.

La migracion `0001_extensiones_enums.sql` ya cubre enums para:

- `biological_sex`
- `lesion_site`
- `case_status`
- `suspicion_level`
- `urgency_level`
- `image_quality_status`
- `review_decision`
- `case_token_purpose`

### Validaciones cubiertas por Edge Functions

`create-case` valida:

- Metodo `POST`.
- JSON obligatorio.
- Campos extra rechazados.
- Consentimiento aceptado.
- Version de consentimiento.
- Edad.
- Sexo declarado.
- Zona bucal.
- Duracion de lesion.
- Ciudad y zona con limite de longitud.

`submit-questionnaire` valida:

- Token o usuario interno.
- Consentimiento existente.
- Cuestionario no duplicado.
- Campos booleanos obligatorios.
- Notas maximo 1000 caracteres.
- Calculo de `risk_score`.

`request-image-upload` valida:

- Estado permitido: `questionnaire_completed` o `image_rejected`.
- MIME permitido: `image/jpeg`, `image/png`, `image/webp`.
- Tamano mayor a 0 y maximo 10 MB.
- Fuente de captura: `camera` o `gallery`.

`finalize-image-upload` valida:

- Estado requerido: `image_upload_requested`.
- UUID de imagen.
- Existencia real del objeto en Storage.
- Imagen original.
- Hash SHA-256 si se envia.
- Dimensiones positivas si se envian.

`validate-image` valida:

- Estado permitido: `image_uploaded`.
- Imagen original.
- MIME permitido.
- Resolucion minima recomendada: 640x480.
- Estado final `quality_accepted` o `image_rejected`.

`run-inference` valida:

- Estado permitido: `quality_accepted`.
- Imagen original.
- Ultima calidad aceptada.
- Modelo activo requerido.
- Servicio IA configurado requerido.
- Respuesta IA con clases `low`, `moderate`, `high`.
- Probabilidades entre 0 y 1.
- Suma de probabilidades entre 0.95 y 1.05.
- Coincidencia de modelo respondido si el servicio envia nombre/version.

`generate-report` valida:

- Estado permitido: `recommendation_ready`, `under_review` o `reported`.
- Recomendacion preventiva existente.
- PDF privado en bucket de reportes.

`review-case` valida:

- Usuario interno con rol `specialist` o `admin`.
- Usuario activo.
- Caso en estado revisable.
- Decision valida.
- Notas clinicas obligatorias y maximo 5000 caracteres.
- Nivel corregido obligatorio cuando aplica.

`create-signed-read-url` valida:

- Token o usuario interno.
- Tipo de recurso permitido.
- UUID de recurso.
- Recurso asociado al caso.
- URL firmada temporal.

`admin-upsert-ai-model` valida:

- Solo rol `admin`.
- Usuario activo.
- Nombre, version y arquitectura.
- `input_shape`.
- `class_labels`.
- `threshold_config`.
- `metrics`.
- `is_active`.

`cleanup-expired-case-tokens` registra auditoria de limpieza y no expone datos sensibles.
